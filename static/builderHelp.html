Custom goal usage

Custom goals are used to define how you want to build your unit. Each custom goal contains one or more [formulae], separated by semicolons [;], without a semicolon at the end. Custom goals can be as simple as a single [EXPRESSION] using a single [VARIABLE], as complex as dealing damage while self-sustaining mp, or even as complex as to be immune to death, elements, and status ailments, while being self-sustaining. Be forewarned, however, that the more unique variables that are included in a custom goal, the more time it will take to calculate the build, or the higher the risk you run of freezing up your device. Reducing the number of cores in use can help prevent freezes, but increases the time needed to calculate the build. The first [formula] is an [EXPRESSION] that provides the builder with it's overall goal; what to maximize, while meeting any following conditions. This [EXPRESSION] is how the builder decides that one build is better than another. [EXPRESSIONS] may contain one or more [variables], using multiplication [*], division [/], or addition [+] to create a single overall value. However, they may not contain the comparison operator [>], which will cause an error. Valid examples: {HP}, {HP * DEF}, {2.6 * P_DAMAGE}. Invalid examples: {HP;}, {HP * SPR > 1} Each [FORMULA] after the first is a [CONDITION] or a [SHORTCUT] for a condition. Each [CONDITION] must compare an [expression] to a specific value; [expression > VALUE]. The [>] operates as [>=], making the [VALUE] provided the minimum goal to satisfy the condition. The right side of the [CONDITION] must be a number. [SHORTCUTS] contain one or more sets [CONDITIONS] with fixed [expression > VALUE] pairs, and cannot be part of a [CONDITION]. For example, the [CONDITION] {R_CONFUSION > 100} is the same as the [SHORTCUT] {I_CONFUSION}. Valid examples: {M_DAMAGE; HP > 6000}, {P_DAMAGE; MP * MP_REFRESH > 45}, {H_DAMAGE; I_PHYSICAL; R_FIRE > 200} Invalid examples: {HP; SPR}, {MP; MAG > 1;}, {MP; M_DAMAGE > 1 + 1}, {DEF; DEF > SPR}, {SPR; R_THUNDER * I_FIRE > 1}

Simulating unit skills

Unit skill multipliers

Unit Skill Multiplers: Skill Multipliers can be entered directly into the custom goal as a multiplier of the damage variable. For example, to calculate damage using a physical attack with a 2.6x multiplier, you would use {2.6 * p_damage}.

Unit buffs and breaks

Unit Buffs and Breaks: Unit Buffs and Breaks are entered into the unit's information, in the third box next to the affected attribute. Buffs and Breaks stack additively, but only the strongest of each is applied. A 120% buff does not gain anything from applying a 50% buff, but does lose some potency to a 50% break, which results in a 70% buff. To calculate this combination of Buff and Break as part of Noctis' Atk, we enter this number (70%) next to his Atk attribute, so that his unit information reads as [atk : 158 (base) : +30 (pots) : 70% (buff)].
