How to use this:

  1. Select a unit you want to make a build for
  2. If you want to make a build only using the items you own, make sure you're logged in, so that your inventory is loaded
  3. If you plan to use an attack with innate element(s), you can select the corresponding element(s) in the "Skill innate element(s)" section
  4. Fill information on the monster you're facing, its race(s) and its elemental resistance (negative number for elemental weakness). Take into account the elemental debufs you'll give it (for instance, if using Orlandeau's Divine Ruination, substract 50 to the light elemental resistance).
  5. Click on "Build !" button to start the computation

Some features are still missing for now. It wont automatically use Equip X materia (but you can pin one to use it)

Custom Formula

How to use this:

  • A formula contains 1 or more EXPRESSION, seprated by semicolons ; (without semicolon after the last one)
  • The first EXPRESSION is used to note a build in order to be able to compare builds. It can uses the VARIABLES and the multiply *, divide / or add + operators between them
  • Any EXPRESSION after the first is a CONDITION. CONDITION must be in the form VARIABLE > VALUE, for instance : HP > 6000
  • Some valid formula :
    HP → Maximize HP
    DEF * HP → Maximize the product of HP by DEF (physical eHP)
    MP * MP_REFRESH → Maximize the number of MP regained each turn
    M_DAMAGE; HP > 6000 → Maximize Magical Damage while keeping HP > 6000
    P_DAMAGE; P_EVADE > 100; HP > 5000 → Maximize Physical Damage while reaching 100% physical evade and HP > 5000
  • The more different variables you use in your formula, the more time the calculations will take
  • Available variables :
    HP, MP, ATK, DEF, MAG, SPRValue of the corresponding basic stat
    P_DAMAGEThis is the total physical damage, taking into account the ATK, the elements, the killers, the damage variance, ...
    M_DAMAGESame as P_DAMAGE but for magical damage
    H_DAMAGESame as P_DAMAGE but for hybrid damage
    M_P_DAMAGESame as P_DAMAGE but for magical damage with physical mechanism
    P_EVADEPhysical evade
    M_EVADEMagical evade
    MP_REFRESH% of MP refresh per turn
    R_FIRE, R_ICE, R_THUNDER, R_WATER, R_EARTH, R_WIND, R_LIGHT, R_DARKCorresponding elemental resistance
    R_POISON, R_BLIND, R_SLEEP, R_SILENCE, R_PARALYSIS, R_CONFUSION, R_DISEASE, R_PETRIFICATION, R_DEATHCorresponding ailment resistance
